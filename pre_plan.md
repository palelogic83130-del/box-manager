# 📦 박스매니저 (Box Manager) 서비스 기획안

## 1. 서비스 개요

* **목적:** 실물 보관 상자에 부착된 QR 코드를 통해 상자 속 물품 리스트를 디지털로 관리.
* **타겟:** 이사 짐 정리, 창고 관리, 수집가(포토카드, 레고 등) 등 대량의 상자를 관리하는 개인 유저.
* **환경:** 모바일 웹 전용 (GitHub Pages 또는 Netlify 배포), No-DB (IndexedDB 기반).

---

## 2. 주요 기능 및 프로세스

### ① 상자 식별 번호 및 QR 전략

* **식별 번호:** **순수 숫자(Positive Integer)**만 사용 (예: 1, 2, 105 등).
* **제약:** 중복 불허 (Unique Key)
* **편의:** 새 상자 추가 시 자동으로 다음 사용 가능한 숫자 추천
* *기획 의도:* 모바일 키보드 전환 없이 숫자 패드로만 빠르게 입력 및 검색 가능.


* **QR 코드 구조:** `https://[도메인]/?id=[숫자]`
* **스마트 스캔:** 앱 내 스캐너 사용 시, URL 전체가 아닌 마지막 `id` 값(숫자)만 추출하여 도메인이 바뀌어도 데이터 매칭 가능.



### ② 카드 템플릿 (출력용)

* **규격:** 표준 포토카드 슬리브 사이즈 대응 (87x56mm 가로형 또는 56x87mm 세로형).
* **디자인:**
    * **테두리:** 지정된 색상 팔레트(파랑, 빨강, 초록, 주황, 보라 등) 중 선택된 색상 적용. 두께는 가독성을 위해 **약 4mm**로 두껍게 처리.
    * **구성:**
        * **가로형 (Landscape):** 좌측에 **QR 코드**, 우측에 크게 적힌 **상자 식별 번호**.
        * **세로형 (Portrait):** 상단에 **QR 코드**, 하단에 크게 적힌 **상자 식별 번호**.
    * **텍스트:** 불필요한 설명 문구 제거.

* **출력:** 
    * 상자 목록에서 출력할 상자를 리스트 형태로 선택.
    * 색상별 필터 기능을 제공하여 특정 카테고리만 모아서 출력 가능.
    * A4 한 장에 다수의 카드를 배치한 PDF 생성.

### ③ 데이터 관리 (No-DB 전략)

* **저장소:** 브라우저 내 **IndexedDB** 사용 (LocalStorage보다 대용량 및 구조화에 유리).
* **백업/복구 (JSON):** * 데이터 유실 방지를 위해 전체 데이터를 `.json` 파일로 내보내기/불러오기 지원.
* 기존 데이터와 병합(Merge) 또는 덮어쓰기(Overwrite) 선택 가능.
* 시크릿 모드 확인 시 화면 상단 '휘발성 주의' 경고 배너 상시 노출.

## 3. 화면 구조 (IA)

| 화면명             | 핵심 기능                                                 | 비고                               |
| ------------------ | --------------------------------------------------------- | ---------------------------------- |
| **메인 (Home)**    | 물건 이름 검색, 최근 본 상자, **[스캔하기]** 플로팅 버튼  | 검색 시 해당 숫자의 상자 위치 노출 |
| **스캔 (Scanner)** | 카메라 권한 승인 후 QR 인식                               | 인식 즉시 상자 상세 화면으로 이동  |
| **상자 상세**      | 해당 상자의 물품 리스트 (추가/수정/삭제)                  | 숫자 번호 크게 상단 노출           |
| **카드 생성/출력** | 숫자 범위 지정(예: 1~9번), 테두리 색상 선택, PDF 다운로드 | A4 레이아웃 미리보기 제공          |
| **설정**           | 데이터 백업(JSON), 전체 초기화, 서비스 정보               | PWA 설치 안내 포함                 |

### 3-1. QR 생성 옵션 기획 (유저 선택권)
출력 화면에서 다음과 같은 옵션을 체크박스로 제공해 보세요.
[ ] 앱 주소 포함 (권장): 카메라 스캔 시 바로 웹사이트로 연결됩니다.
[ ] 고유 번호만 포함: 오직 박스매니저 앱 내부 스캔으로만 동작하며 보안성이 높습니다.
[ ] 텍스트 병기: QR 옆에 적히는 번호를 크게/작게 조절하거나 숨길 수 있는 옵션.

## 4. 유저 시나리오 (UX)

1. **준비:** 앱에서 상자 번호(1~9번) 선택 후 테두리 색상 지정 -> PDF 생성 및 출력.
2. **부착:** 출력물을 잘라 상자에 붙인 슬리브에 삽입.
3. **등록:** 상자에 물건을 넣을 때 앱의 **[스캔]** 버튼 클릭 -> QR 촬영 -> 물건 이름 입력(예: "겨울 니트", "공구 세트").
4. **조회:** 나중에 물건을 찾을 때 홈 화면에서 "공구" 검색 -> **"5번 상자(빨간색)"** 결과 확인.
5. **이전:** 폰을 바꿀 때 이전 폰에서 **[JSON 내보내기]** -> 새 폰에서 **[불러오기]**.
